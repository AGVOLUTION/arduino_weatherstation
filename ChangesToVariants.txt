Problem Description:
We need to re-route the Serial1 (SERCOM5) USART interface.
Normally, Serial1 is connected to Board-Pins 13/14 (RX/TX).

We would like to use the TX line for debug output (printf-debug).
We would like to use the RX line for GPS module reception (9600 baud normally).

#define PIN_SERIAL1_RX (13ul) -> #define PIN_SERIAL1_RX (16ul) // from Pin 13 to 16 (A1)
#define PAD_SERIAL1_RX (SERCOM_RX_PAD_3) -> #define PAD_SERIAL1_RX (SERCOM_RX_PAD_0)
